# apbd-cw1-git-s33233



The merger could not be done by fast forward, as the main branch had diverged and git cannot simply change the head to the latest commit.

1. Git performs fast forwards when there is no divergence in its branches, allowing the head to simply be moved to the latest commit. Similarly, a merge commit is created when the opposite is true, and there is a divergence that needs to be converged to unify two branches.

2. A merge is the union of two different branches, while rebasing is the re-linearization of the tree by "sliding" a branch forward to the head of the target branch.

3. Both lines were selected, as they were not mutually exclusive by nature, and kept, preserving both the changes that were made.
